[주요 목표]
블루프린트와 c++를 활용하여 LostArk 게임의 보스레이드 모작을 진행하나 블루프린트를 활용하면 효율적인 부분에는 블루프린트, c++을 활용하면 효율적인 부분에는 c++를 적용시켜
만드는것이 목표. 다만, Lyra의 프레임워크를 따라갈것임. Lyra에서 가져온 플러그인으로 Game Ability System, Modular Gameplay, Modular Gameplay Actors, CommonUI, UIExtension,
CommonGame, CommonUser, GamePlayMessageRouter가 있음. gamefeature의 경우 이번프로젝트에서는 적용하지않으나 Lyra의 철학인 모듈화는 따라감. Experience는 프로젝트에
적용. 원작의 경우 4인 멀티 파티로 진행하는게 일반적이나 이번 프로젝트에서는 싱글로 진행. (보스와 플레이어의 1:1)

[구현계획]
1. UMyPawnData (DataAsset)
 * 역할: "이 캐릭터는 어떤 Pawn 클래스이고, 어떤 스킬과 입력 설정을 쓰는가?"
 * 이동할 데이터:
     * PawnClass (BP_Kwang)
     * StartupAbilities (기존 Character에 있던 것)
     * InputConfig (기존 Character에 있던 것)

2. UMyExperienceDefinition (DataAsset)
 * 역할: "이 게임 모드는 어떤 PawnData를 기본으로 쓰는가?"
 * 포함할 데이터:
     * DefaultPawnData (UMyPawnData 포인터)
     * (나중에) HUDClass, ActionSets 등

3. UMyExperienceManagerComponent (GameStateComponent)
 * 역할: 게임 시작 시 지정된 ExperienceDefinition을 로드하고, 다른 클래스들이 "지금 무슨 모드야?"라고 물어보면 알려줍니다.

4. UMyHeroComponent (PawnComponent)
 * 역할: 캐릭터가 스폰되면 ExperienceManager에게 PawnData를 달라고 요청한 뒤, GAS 초기화(InitAbilityActorInfo) + 스킬 부여(GrantAbility) + 입력 바인딩(BindInput)을 수행합니다.
 * 결과: 기존 AMyCPPProjectCharacter에 있던 복잡한 로직들이 싹 사라지고 이 컴포넌트로 이동합니다.


📅 오늘 작업 요약 (2주차: Experience 시스템 구축)

1. 데이터 에셋 구조화 (Data Driven)
 * `UMyPawnData`: 캐릭터가 사용할 스킬(Abilities)과 입력 설정(InputConfig), 입력 매핑(IMC)을 하나의 데이터 파일로 묶었습니다.
 * `UMyExperienceDefinition`: 게임 모드별로 어떤 PawnData를 쓸지 정의하는 설정 파일을 만들었습니다.
 * 결과: 코드를 수정하지 않고도 에디터에서 DA_KwangPawnData만 수정하면 스킬이나 입력을 마음대로 바꿀 수 있습니다.

2. 매니저 시스템 (Manager Component)
 * `UMyExperienceManagerComponent`: GameState에 부착되어 현재 로딩된 Experience 데이터를 관리하고, 필요한 곳(Character 등)에 알려주는 방송국 역할을 합니다.
 * `AMyCPPProjectGameState`: 위 컴포넌트를 담을 그릇(GameState)을 새로 만들었습니다. (이 부분이 연결의 핵심이었습니다!)

3. 히어로 컴포넌트 (Hero Component)
 * `UMyHeroComponent`: 캐릭터(Character)의 두뇌 역할을 합니다.
     * 비동기 로딩: 게임 시작 시 데이터가 준비될 때까지 기다렸다가, 준비되면 즉시 초기화를 수행합니다.
     * GAS 초기화: PawnData에 있는 스킬들을 ASC에 등록합니다.
     * 입력 바인딩: 입력 태그(Input Tag)를 사용하여 키보드 입력과 스킬을 1:1로 연결했습니다.

4. 리팩토링 및 버그 수정
 * Character 다이어트: AMyCPPProjectCharacter에 하드코딩되어 있던 스킬/입력 로직을 모두 제거하고 HeroComponent로 위임했습니다.
 * ASC 연결: PlayerState에 있는 ASC를 가져와 캐릭터와 연결(InitAbilityActorInfo)하여 GAS가 정상 작동하도록 고쳤습니다.


내일(2.5주차 Day 1) 작업 계획

목표: 캐릭터 이동 복구 및 GAS 전투 시스템(Cost/Cooldown) 기초 구현

1. 캐릭터 기본 조작(Move/Look) 복구
 * 현재 문제: HeroComponent로 입력 권한을 넘기면서, 기존 Character 클래스에 있던 이동(WASD) 및 시점 조작 코드가 연결되지 않음.
 * 작업 내용:
     * MyHeroComponent에 Input_Move, Input_Look 함수 구현.
     * PawnData의 NativeInputActions를 순회하며 해당 함수들을 바인딩.
     * 결과적으로 캐릭터가 다시 걷고, 카메라를 돌릴 수 있게 만듦.

2. GAS 기초: Cost & Cooldown (마나 & 쿨타임)
 * 개념: 스킬을 무한정 쓰지 못하게 제약 조건을 겁니다.
 * 작업 내용:
     * AttributeSet: 마나(Mana) 속성 추가.
     * GameplayEffect:
         * GE_ManaCost: 마나를 깎는 이펙트.
         * GE_Cooldown: 스킬 사용 후 일정 시간 동안 '쿨타임 태그'를 부여하는 이펙트.
     * GameplayAbility: GA_Skill_Q에 위 이펙트들을 연결하여 "마나가 있고 쿨타임이 아닐 때만 발동"되도록 설정.

3. (시간 남으면) 평타 콤보 시스템
 * 개념: 클릭할 때마다 모션이 바뀌는(휘두르기 -> 찌르기 -> 마무리) 콤보 공격.
 * 작업 내용: Wait Input Press 등의 GAS Task를 활용한 분기 처리.
